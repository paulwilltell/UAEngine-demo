### Full Description (for the listing)
```
## The Problem

Every AI gives the same response to every user. Frustrated developer gets the same wall of text as a curious beginner. Expert in a rush gets the same explanation as someone who's overwhelmed.

Generic responses = frustrated users = churn.

## The Solution

UAEngine is a lightweight middleware that sits between your users and your LLM. It analyzes **how** users write:

- Punctuation patterns ("???" vs "!!!")
- Message pacing and rhythm
- Cognitive load signals
- Likely follow-up intent

Then it shapes your AI's response accordingly:
- Frustrated user? ‚Üí Lead with the fix, skip the explanation
- Beginner? ‚Üí Use analogies, avoid jargon
- Expert in a rush? ‚Üí 3 lines max, code first
- Overwhelmed? ‚Üí One concept at a time, numbered steps

## What UAEngine Is NOT

‚ùå Not emotion detection or psychological diagnosis
‚ùå Not a replacement for your AI model
‚ùå Not "mind reading" ‚Äî we analyze writing patterns, not mental states

## How It Works

1. User sends a message
2. UAEngine extracts signals (47ms latency)
3. Your app receives structured adaptation hints
4. You shape the system prompt accordingly
5. AI responds in the right style for that user

## Built For

- Customer support bots
- EdTech platforms
- Developer tools & documentation
- Any conversational AI that wants to feel less robotic
```
