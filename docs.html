Configure the UAEngine client with optional settings.

Python 
client = uaengine.Client(
    api_key="ua_live_sk_your_api_key",
    
    # Optional configuration
    timeout=30,           # Request timeout in seconds
    max_retries=3,        # Retry failed requests
    base_url="https://api.uaengine.ai",  # Custom endpoint (enterprise)
)
Option Type Default Description
api_key string Required Your UAEngine API key
timeout int 30 Request timeout in seconds
max_retries int 3 Number of retry attempts
base_url string api.uaengine.ai API endpoint (for enterprise)

üìä Signal Analysis

UAEngine analyzes 5 distinct signals from user communication to help your AI respond more effectively.

Signal Name What It Detects
MEP Message Emotional Profile Emotional intensity and sentiment (0.0 - 1.0)
CLE Cognitive Load Estimation User's mental state (relaxed, focused, overloaded)
ARM Adaptive Response Mode Recommended response style
SIQ Session Intelligence Quotient Cross-message pattern analysis
TLP Temporal Learning Profile Learning curve and comprehension speed

üé≠ MEP Score

The Message Emotional Profile score indicates the emotional intensity of a message on a scale from 0.0 to 1.0.

Score Range Emotional State Recommended Approach
0.0 - 0.3 Calm / Neutral Normal, informative responses
0.3 - 0.6 Engaged / Curious Detailed, enthusiastic responses
0.6 - 0.8 Frustrated / Concerned Empathetic, solution-focused
0.8 - 1.0 Highly Emotional Acknowledge feelings first, then assist
Example Response 
{
  "mep": {
    "score": 0.87,
    "primary_emotion": "frustration",
    "secondary_emotion": "confusion",
    "confidence": 0.94
  }
}

üß† CLE Analysis

The Cognitive Load Estimation determines how much mental bandwidth a user has available.

State Indicators Response Strategy
relaxed Complete sentences, clear questions Full explanations welcome
focused Specific questions, technical terms Precise, detailed answers
distracted Typos, incomplete thoughts Clear, structured responses
overloaded Fragmented messages, frustration Simple, step-by-step guidance

üéØ ARM Recommendations

The Adaptive Response Mode suggests how to structure your AI's response.

Mode When Used Response Pattern
tldr_expand User needs quick answer Answer first, details after
step_by_step Complex task, user overwhelmed Numbered steps, one at a time
deep_dive User wants to understand why Detailed explanation
confirm_first Ambiguous request Clarify before proceeding
empathize_solve Frustrated user Acknowledge, then solve

üì° analyze()

The core method for analyzing a single user message.

Python 
result = client.analyze(
    message="How do I implement OAuth?",
    
    # Optional parameters
    context="User is building a web app",
    session_id="user_123_session_456",
    include_signals=["mep", "cle", "arm"]
)

Parameters

Parameter Type Required Description
message string Yes The user message to analyze
context string No Additional context about the conversation
session_id string No ID for multi-message session analysis
include_signals array No Specific signals to return (default: all)

Response

JSON Response

  "id": "ana_abc123",

üí° Examples

Customer Support Bot

Python 
def handle_support_message(user_message):
    # Analyze the message
    analysis = client.analyze(message=user_message)
    
    # Adapt response based on signals
    if analysis.mep.score > 0.7:
        # User is frustrated - acknowledge first
        prefix = "I understand this is frustrating. Let me help you right away.\n\n"
    else:
        prefix = ""
    
    # Generate response with AI
    response = generate_ai_response(
        message=user_message,
        tone=analysis.adaptation.tone,
        structure=analysis.arm
    )
    
    return prefix + response

def teach_concept(question, session_id):
    analysis = client.analyze(
        message=question,
        session_id=session_id
    )

        # Simplify explanation
